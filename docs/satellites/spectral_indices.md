# Spectral Indices

## Overview

To streamline the process of producing Spectral Index bands when downloading images, the GeeFetch configuration allows the user to add a list of spectral index names under the **spectral_indices** attribute to add as bands of the downloaded data.

All indices implemented were pulled from the curated package [awesome-spectral-indices](https://github.com/awesome-spectral-indices/awesome-spectral-indices.git). This is a sub-selection of the indcies implemented that match the sensors available in GeeFetch.

The implementation was made specifically for the GeeFetch package; we avoided adding [eemont](https://github.com/davemlz/eemont/tree/master) and [ee_extra](https://github.com/r-earthengine/ee_extra/tree/master) as dependencies. See these packages for a more global integration of spectral indices to ee.

### Command Line

Write the following `config.yaml`

```yaml
data_dir: ~/satellite_data
satellite_default:
  aoi:
    spatial:
      left: 2.2
      bottom: 48.7
      right: 2.5
      top: 49
      epsg: 4326
    temporal:
      start_date: "2023-06-01"
      end_date: "2023-06-30"
  gee:
    ee_project_ids: ["your-gee-id"]
  tile_size: 2000
  resolution: 10
s2: # more scrict cloud filtering than the defaults
  spectral_indices:
    - NDVI
    - NBR
s1:
  spectral_indices:
    - QpRVI
    - RFDI
palsar2:
  spectral_indices:
    - RFDI
landsat8:
  spectral_indices:
    - BITM
    - BIXS
    - RI4XS
```

then download with

```bash
geefetch s1 -c config.yaml
geefetch s2 -c config.yaml
geefetch palsar2 -c config.yaml
geefetch landsat8 -c config.yaml
```
